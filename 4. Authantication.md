## Authentication
Applications must authenticate before accessing Guardian IDM APIs. Successful authentication generates a valid access token. Access tokens enable secure access to Guardian IDM services. Administrators and integrators must generate and refresh access tokens.
|Capability|Endpoint|
|----|----|
|Generate access token|POST/api/auth/token?grant_type=password|
|Refresh access token|POST/api/auth/token?grant_type=refresh_token&refresh_token={{refresh_token}}|

**Request Example - Generate access token**

```POST /api/auth/token?grant_type=password HTTP/1.1
Host:
Content-Length: 59

{
  "username": "admin"
  "password": "Alert1234"
}
```
